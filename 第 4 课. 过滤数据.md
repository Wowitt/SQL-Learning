### 4.1 使用 `WHERE` 子句

`WHERE condition`

> *SQL* 过滤与应用过滤
>
> 合适的 *SQL* 总是能提供更高的性能和速度。如果过滤操作被编码进应用，会使得应用不具备可伸缩性。

### 4.2 WHERE 子句操作符

| 操作符 | 说明 |
| :------: | :----------------- |
| `=`      | 等于               |
| `<>` 或 `!=` | 不等于             |
| `<`      | 小于               |
| `<=`     | 小于等于           |
| `!<`     | 不小于             |
| `>`      | 大于               |
| `>=`     | 大于等于           |
| `!>`     | 不大于             |
| `BETWEEN` | 在指定的两个值之间 |
| `IS NULL` | 为 `NULL` 值       |

具体的可用操作符需要参阅 *DBMS* 的具体文档。

#### 4.2.1 检查单个值

`WHERE col OP value`

#### 4.2.2 不匹配检查

`WHERE col <> value`

#### 4.2.3 范围值检查

`WHERE col BETWEEN val1 AND val2`

#### 4.2.4 空值检查

`WHERE col IS NULL`
